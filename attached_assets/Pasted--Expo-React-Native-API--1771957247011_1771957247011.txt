너는 내 Expo (React Native) 앱 프로젝트에 “한국 식약처 공공데이터 API 기반 약 정보/상호작용 안내 기능”을 추가하는 역할이야.
목표: 사용자가 약 이름(예: 오메가3, 타이레놀 등)을 입력하면,
1) 식약처 ‘의약품개요정보(e약은요)’ API에서 해당 약의 효능/사용법/주의사항/상호작용/부작용/보관법을 가져와 요약해서 보여주고,
2) (가능하면) 식약처 DUR(의약품안전사용서비스) 데이터로 병용금기/연령금기/임부금기 등 “안전 경고”를 추가로 표시한다.
3) UI는 과하게 복잡하지 않게, ‘요약 카드 + 펼치기(accordion)’ 형태로 구성한다.
4) 서비스키(ServiceKey)는 절대 클라이언트(앱) 코드에 하드코딩하지 말고, Replit 서버(Express)로 프록시를 만들어 환경변수로 숨긴다.

=== 필수 요구사항 ===
A) 백엔드(프록시 서버) 추가
- /server/index.js (또는 기존 서버 파일)에 Express 서버를 만든다.
- 환경변수: DATA_GO_KR_SERVICE_KEY 를 사용한다.
- 엔드포인트 1: GET /api/mfds/easy-drug?itemName=...&pageNo=1&numOfRows=10
  -> 내부에서 아래 공공데이터포털 API를 호출해 JSON으로 반환한다.
  -> 호출 URL (식약처 e약은요): http://apis.data.go.kr/1471000/DrbEasyDrugInfoService/getDrbEasyDrugList
  -> 파라미터: ServiceKey, itemName, pageNo, numOfRows, type=json(가능하면)
  -> 에러/빈 결과 처리(“검색 결과 없음” 메시지)

- 엔드포인트 2(선택이지만 가능하면 구현): GET /api/mfds/dur?query=... (혹은 itemName 기반)
  -> DUR API는 data.go.kr의 ‘식품의약품안전처_의약품안전사용서비스(DUR)품목정보’ 문서/Swagger를 참고해서 호출한다.
  -> 내가 나중에 정확한 DUR 호출 URL/파라미터를 확정할 수 있게, 코드를 “구성 가능”하게 작성한다:
     - DUR_BASE_URL, DUR_PATH 등을 상수/환경변수로 분리
     - 지금은 placeholder로 두되, 동작 구조(프록시/응답 포맷/에러 처리)는 미리 완성

- 서버는 CORS 설정을 해주고, 클라이언트에서 호출 가능하게 한다.

B) 프론트엔드(UI) 추가
- 새 화면: “약 정보(Drug Info)” 화면 추가
  1) 검색 입력창(약 이름)
  2) 검색 버튼
  3) 로딩 스피너
  4) 결과 표시: 카드 섹션 구성
     - 상단: 제품명/업체명 (가능하면)
     - 섹션(accordion): 효능(efcyQesitm), 사용법(useMethodQesitm), 주의사항경고(atpnWarnQesitm), 주의사항(atpnQesitm), 상호작용(intrcQesitm), 부작용(seQesitm), 보관법(depositMethodQesitm)
  5) “출처 표기”를 화면 하단에 고정: “출처: 식품의약품안전처 공공데이터(e약은요)”라고 명시
  6) “의학적 조언 아님(면책)” 문구를 작은 글씨로 표시: “본 정보는 참고용이며, 복약 관련 결정은 의료 전문가와 상담하세요.”

- 앱 내 약 등록 흐름과 연결:
  - 약 등록할 때 ‘정보 보기’ 버튼을 누르면 이 Drug Info 화면으로 넘어가고, 입력된 약 이름을 자동으로 검색한다.

C) 데이터 가공(간단 요약)
- 긴 텍스트는 그대로 다 보여주면 부담스러우니:
  - 기본은 3~5줄 미리보기 + “더 보기”로 확장
  - 혹은 bullet 요약(문장 단위로 나눠 앞부분 몇 개만)
- 한국어 텍스트 줄바꿈/특수문자 처리 잘하기.

D) 구현 품질
- TypeScript면 TS로, JS면 JS로 일관되게 유지
- 네트워크 호출은 fetch 또는 axios 하나로 통일
- API 응답이 XML로 올 경우를 대비해, 서버에서 JSON 변환 또는 type=json 파라미터를 사용
- 코드에 주석으로 “여기서 DUR 엔드포인트 확정되면 이 부분 수정” 표시
