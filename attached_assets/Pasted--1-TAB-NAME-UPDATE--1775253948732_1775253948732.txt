Update the sharing/connection model to be approval-based and directional.

Important:
Entering an invite code must NOT create an immediate connection.

New required behavior:
1. If User A enters User B’s invite code, this should create a pending request only.
2. User B (the invite code owner / primary user) must receive a request and explicitly approve it.
3. Only after approval should User A be allowed to view User B’s medication status.

Please model this as a directed permission system, not an undirected friendship system.

Meaning:
- A may be allowed to view B
- B does NOT automatically gain permission to view A

This app should support asymmetric relationships such as:
- child can view mother
- mother does not automatically view child

Please update the UI accordingly.

In the sharing tab, show separate sections:
1. People I can view
2. People who can view me

Each section should show clear status:
- accepted
- pending
- rejected
- revoked / removed

Examples:
- People I can view:
  - Mom — accepted
  - Dad — pending approval
- People who can view me:
  - Sister — accepted
  - Friend — request pending

Please make the UX feel like permission and consent management, not generic social connection.

- “OO님이 회원님의 복용 상태를 보려고 합니다”
- “승인 후 함께 보기에서 확인할 수 있습니다”
- “내 상태를 볼 수 있는 사람”
- “내가 보고 있는 사람”

Please update the backend/data model as needed to support:
- owner_user_id
- viewer_user_id
- request status
- created_at / approved_at
- optional relationship label
